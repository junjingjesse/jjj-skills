# 服务设计需求分析方法（增强版）

## 使用流程

**第1步**：用户简述想法/需求（1-2句话）

**第2步**：你提问访谈（5-10个关键问题）

**第3步**：联网查询拓展（可选）
- 查行业现状
- 查竞品案例
- 查用户评价/痛点

**第4步**：输出完整分析
- 需求判断
- 顾客旅程地图
- 服务蓝图
- 创新机会点
- 风险评估
- 商业可行性

---

## 第一步：用户初步描述

用户需要提供（至少）：
- 你想做什么产品/服务？
- 给谁用的？（目标用户）
- 解决什么问题？

---

## 第二步：访谈式提问（你问用户）

### 基础问题（必问）

1. **目标用户画像**
   - 谁会用这个？年龄/收入/场景
   - 他们现在怎么解决这个问题的？

2. **需求真实性**
   - 用户真的需要这个吗？愿意付费吗？
   - 频率如何？（高频/低频）

3. **竞争情况**
   - 有没有竞品？他们哪里做得不好？
   - 你有什么不一样？

4. **交付形式**
   - 线上还是线下？
   - 卖产品还是卖服务？

5. **盈利假设**
   - 想怎么赚钱？
   - 用户愿付多少钱？

### 进阶问题（根据情况问）

6. **场景还原**
   - 用户在什么场景下会用？
   - 使用前做什么？使用后做什么？

7. **痛点深挖**
   - 用户最痛的点是什麼？
   - 他们现在用什么替代？满意吗？

8. **资源假设**
   - 你有什么资源能做这个？
   - 需要多少人/多少钱/什么技术？

---

## 第三步：联网查询拓展（你说要做，然后执行）

### 查询类型

**1. 行业现状查询**
- 这个领域的市场规模、趋势
- 主要玩家有谁
- 行业痛点是什么

**查询指令示例**：
```
使用 web_access 搜索：[你的产品类型] 行业分析/市场报告
使用 web_access 搜索：[你的领域] 2024 2025 趋势
```

**2. 竞品案例查询**
- 竞品有哪些？做得怎么样？
- 用户对他们评价如何？差评是什么？

**查询指令示例**：
```
使用 web_access 搜索：[你的产品类型] + 竞品分析
使用 web_access 搜索：[产品名] 用户评价 优缺点
```

**3. 用户痛点查询**
- 用户真正不满的是什么？
- 现有方案的坑是什么？

**查询指令示例**：
```
使用 web_access 搜索：[你的产品类型] + 痛点 + 用户抱怨
使用 web_access 搜索：[你的领域] + 常见问题
```

**4. 创新案例查询**
- 类似创新案例有哪些？
- 他们怎么成功的？

**查询指令示例**：
```
使用 web_access 搜索：[你的领域] 创新案例 创业
使用 web_access 搜索：[你的产品类型] 成功案例
```

### 联网查询输出格式

查完后简要总结：
```
行业现状：
- 市场规模：XXX
- 主要玩家：XXX
- 趋势：XXX

竞品分析：
- 竞品A：XXX（优/劣）
- 竞品B：XXX（优/劣）

用户痛点：
- 用户不满点1：XXX
- 用户不满点2：XXX

创新机会：
- 机会点1：XXX
- 机会点2：XXX
```

## 第四步：输出完整分析

### 1. 需求真假判断（必答）

| 指标 | 判断 | 根据 |
|-----|------|------|
| 需求真不真 | 真/假/待验证 | 用户访谈+联网查询 |
| 需求够不够大 | TAM/SAM/SOM | 行业数据 |
| 需求稳不稳定 | 昙花一现/长期 | 趋势判断 |
| 付费意愿强不强 | 强/中/弱 | - |

### 2. 顾客是谁（1-2句话）

- 核心用户画像
- 他们共同特征

### 3. Job to be Done（1句话）

顾客使用这个产品真正要完成什么任务？

### 4. 顾客痛点深挖

#### 4.1 四种顾客犧牲类型

| 犧牲类型 | 特征 | 你的产品可能对应 |
|---------|------|---------------|
| 太难取舍 | 功能vs外观、选项太多 | 协作式客制化 |
| 漫长筛选过程 | 难以判断适合与否 | 适应式客制化 |
| 外形犧牲 | 功能OK但包装不满 | 装饰性客制化 |
| 千篇一律重复 | 反复提交相同信息 | 透明性客制化 |

#### 4.2 痛点强度

- **犧牲**：顾客勉强接受（可用但不满）
- **痛点**：明显负面情绪（皱眉/停顿）
- **爽点**：超出预期的惊喜

#### 4.3 体验四领域

| 领域 | 适合场景 | 你的产品能沾哪个？ |
|-----|---------|-------------------|
| 娱乐 | 被动吸收 | 影视/餐饮 |
| 教育 | 主动吸收 | 培训/知识付费 |
| 逃离现实 | 积极沉浸 | 主题乐园/游戏 |
| 美学 | 消极沉浸 | 艺术展/精品店 |

### 5. 顾客旅程地图

#### 5.1 五问法（绘制前）

| 问题 | 答案 |
|-----|------|
| Who（顾客是谁）| |
| What（目标是什么）| |
| Before（来之前做了什么）| |
| After（之后做什么）| |
| Emotion（情绪如何）| |

#### 5.2 旅程阶段（示例）

| 阶段 | 顾客做什么 | 想什么 | 情绪 | 痛点 |
|-----|-----------|-------|------|------|
| 认知 | ○○ | ○○ | ○○ | - |
| 考虑 | ○○ | ○○ | ○○ | ●● |
| 购买 | ○○ | ○○ | ○○ | ●● |
| 使用 | ○○ | ○○ | ○○ | ●● |
| 推荐 | ○○ | ○○ | ★★ | - |

(●● = 痛点，★★ = 爽点)

#### 5.3 CJM绘制要素

- **Doing**：顾客具体行动
- **Thinking**：顾客在想什么
- **Feeling**：顾客情绪变化
- **Experience**：顾客经历细节
- **Opportunities**：可改进/创新的机会

### 6. 服务蓝图

#### 6.1 三条线

- **Line of Interaction**：顾客与前台交互点
- **Line of Visibility**：前台可见 vs 后台不可见
- **Line of Internal Interaction**：后台与支援系统协作

#### 6.2 蓝图结构

```
[Physical Evidence]
    ↓
[User Actions] ←──────→ [Front Stage]
    ↓
[Line of Visibility]
    ↓
[Back Stage]
    ↓
[Support Process]
```

### 7. 服务方案设计

#### 7.1 四层次维度

1. **Service Operation（服务运作）**
   - 前台接触点
   - 后台支撑

2. **Service Experience（服务体验）**
   - 顾客情绪
   - 满意度

3. **Service Outcome（服务结果）**
   - 达成预期？
   - 超预期→忠诚

4. **Value of the Service（服务价值）**
   - 价格/便利/体验综合衡量

#### 7.2 服务创新四方向

| 方向 | 难度 | 例子 |
|-----|------|------|
| 全新服务 | 高 | 以前没有 |
| 核心服务创新 | 中 | 改变本质 |
| 递送方式创新 | 中 | 物流/渠道 |
| 辅助服务创新 | 低 | APP/售后 |

#### 7.3 V=C×(A+P) 公式

- **C**：经营策略（低成本/差异化/焦点集中）
- **A**：资产（人/设备/技术/资金）
- **P**：流程设计
- **V**：最终价值

### 8. 三大服务策略

| 策略 | 适合场景 | 你的产品适合哪个 |
|-----|---------|----------------|
| 成本优势 | 价格敏感、规模化 | Costco模式 |
| 差异化 | 品质/体验优先 | 苹果模式 |
| 焦点集中 | 小众市场 | Shouldice医院模式 |

### 9. DMAIC流程改善（如需）

- **D**efine：界定问题
- **M**easure：衡量现状
- **A**nalyze：分���原��
- **I**mprove：改进方案
- **C**ontrol：控制成果

### 10. 收益与定价

#### 10.1 能不能差别定价？

| 判断条件 | 是/否 |
|---------|-------|
| 易腐商品？（时效限制）| |
| 需求波动大？（淡旺季）| |
| 固定成本高？（边际成本低）| |

#### 10.2 三级差别定价

| 级别 | 方式 | 你的产品怎么用 |
|-----|------|-------------|
| 第三级 | 团体定价 | 学生票/老人票 |
| 第二级 | 菜单式定价 | 电信方案/运费选项 |
| 第一级 | 完美歧视 | 按人定价（理想）|

#### 10.3 消费者心理

- **期望理论**：「折扣感」比「附加费」更愿意接受
- **心理预算**：钱分不同心理账户
- **公平性**：涨价要能说明理由

#### 10.4 动态定价（Uber参考）

- 一口价（心理冲击小）
- 倍数显示（冲击大）
- 个性化定价（看人下菜）
- 下游调度（激励用户帮移动资源）

### 11. 平台可能性

#### 11.1 能不能做平台？

先问三问题：
1. 是否需要联结多方？
2. 有没有网络外部性？
3. 能不能降低信息不对称？

#### 11.2 网络外部性分析

| 类型 | 正向 | 负向 |
|-----|------|------|
| 同边 | 社交越多人越好 | 司机太多抢单 |
| 跨边 | eBay买家多→卖家来 | 假货多→买家跑 |

#### 11.3 临界人数三招

1. **提升独立价值**：让人没网络也能用
2. **补贴**：烧钱快速拉用户
3. **聚焦**：先做小人群再扩

#### 11.4 交叉补贴设计

谁补贴谁？看两点：
1. **价格弹性**：弹性高的这边免费/低价
2. **同边网路效应**：效应强的这边培育

典型：
- C2C市集：卖家付费→买家免费
- 搜寻引擎：广告商付费→用户免费

### 12. 流程三步曲（优化用）

| 阶段 | 适用 | 例子 |
|-----|------|------|
| 流程设计 | 新业务 | 从0开始设计 |
| 流程改善 | 优化中 | 微调改进 |
| 流程再造 | 颠覆性 | 重新定义 |

### 13. 六标准差六大主轴

1. 以顾客为尊
2. 数据与事实带动管理
3. 流程就是行动
4. 主动管理
5. 无界线协力合作
6. 追求完美与容忍失败

### 14. 同理心设计五类资讯

1. **使用时机**：什么时候用什么
2. **使用情境与环境**：什么场景下用
3. **使用者的自行改造**：用户怎么改造产品
4. **隐性特征**：情感/记忆/联想
5. **未曾察觉的需求**：用户习惯但不说

### 15. 创新案例对照

| 你的需求 | 类似案例 | 参考点 |
|---------|---------|-------|

## 激发问题（分析中追问）

- 「顾客真的在乎这个吗？还是我们在自嗨？」
- 「有没有更简单的做法能解决同样问题？」
- 「这个体验能让人记住吗？」
- 「如果不收费还能做吗？」
- 「线下能做到吗？必须线上？」
- 「顾客的替代方案是什么？为什么选这个不选那个？」
- 「这个需求够不够大？有没有人已经做过了？」
- 「最赚钱的环节是什么？能不能copy到别人？」
- 「用户愿付钱的那个点，我们有没有？」

## 案例：完整服务设计输出示例

（用户说：我想做一个针对上班族的午餐外卖平台）

```markdown
# 完整分析：上班族午餐外卖平台

## 第一部分：需求判断

### 1.1 需求真假
- 判断：真需求（高确定性）
- 理由：外卖用户基数大、频次高、付费意愿已验证

### 1.2 目标用户
- 核心用户：25-40岁上班族
- 场景：工作日午餐、预算20-40元
- 痛点：不知道吃什么、选择困难、等待时间长

### 1.3 Job to be Done
「让上班族在30分钟内吃到合适午餐，不用思考」

### 1.4 市场规模
[联网查询中...]
- 中国外卖市场规模：约1.2万亿
- 白领午餐细分：约2000亿
- 主要玩家：美团、饿了么、抖音外卖

### 1.5 联网查询总结
- 竞品：美团（全面）、饿了么（补贴）、抖音（内容）
- 用户痛点：菜品同质化、选择困难、送达慢
- 机会点：精准推荐、SKU差异化

## 第二部分：痛点分析

### 2.1 顾客犧牲类型
- 太难取舍（选择困难）→协作式客制化
- 千篇一律重复（每天重复选）→透明性客制化

### 2.2 痛点强度
- ★★★ 不知道吃什么（选择困难）
- ★★ 等太久
- ★ 不好吃

### 2.3 体验机会
- 「娱乐」：盲盒玩法
- 「教育」：健康饮食建议

## 第三部分：服务方案

### 3.1 顾客旅程（5阶段）
| 阶段 | 行为 | 情绪 | 痛点 |
|-----|------|------|------|
| 打开App | 犹豫吃什么 | 困惑 | ★★★ |
| 浏览 | 反复刷 | 焦虑 | ★★ |
| 下单 | 等待 | 期待 | - |
| 等候 | 看进度 | 着急 | ★★ |
| 收到 | 开吃 | 满足 | - |

### 3.2 服务蓝图
- 前台：App下单、骑手配送、客服
- 后台：商家管理、算法推荐、骑手调度
- 支援：支付系统、配送地图、评价体系

### 3.3 创新点
1. 盲盒推荐（解决选择困难）
2. 会员订阅（解决重复点）
3. 准时保（解决等待焦虑）
...
```
