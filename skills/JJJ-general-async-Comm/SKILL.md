# JJJ-general-async-comm 异步沟通初始化技能

在指定文件夹创建异步沟通所需的标准文档结构，并提供任务管理流程。

## 整体流程

```
用户触发技能
    ↓
创建 async_comm 文档结构 (USER/TASK/Q&A/REGRESSION/RETROSPECTIVE)
    ↓
创建定时任务 (每5分钟执行)
    ↓
┌─────────────────────────────────┐
│ 定时任务执行 SOP                │
│                                 │
│ 1. 回归测试 → 检查 REGRESSION   │
│ 2. 任务执行 → 从 TASK.md 取任务  │
│ 3. 回归测试 → 检查 RETROSPECTIVE │
│ 4. 复盘沉淀 → 写 RETROSPECTIVE │
│ 5. 归档已完成 → archive/      │
│ 6. 重置TASK占位符            │
│ 7. 输出结果                   │
└─────────────────────────────────┘
经验沉淀在 RETROSPECTIVE.md
```

## 快速开始

用户触发此技能时，直接说：

**请提供要初始化异步沟通的文件夹路径。**

同时告知用户：
> 初始化后，我将创建一个**5分钟一次**的定时任务，自动执行该项目的任务流程。

等待用户输入路径后，创建文档结构并设置定时任务。

---

## 第一步：创建文档结构

在用户提供的路径下创建：
```
路径/
└── async_comm/
    ├── USER.md          ← 用户写需求
    ├── TASK.md        ← 执行者任务
    ├── Q&A.md         ← 疑问记录
    ├── REGRESSION.md  ← 回归测试（功能检查）
    ├── RETROSPECTIVE.md ← 复盘记录（经验沉淀）
    └── archive/         ← 已完成任务归档
        └── 2026-05-06_1430_任务名称.md  ← 年月日_时分_任务概述
```

### archive目录结构（归档文件）

归档文件名格式：`YYYY-MM-DD_HHmm_任务概述.md`

示例：
```
archive/
├── 2026-05-06_1430_完成用户登录功能.md
├── 2026-05-06_1500_修复登录超时问题.md
└── 2026-05-07_0915_添加忘记密码功能.md
```

### 归档流程

1. **识别**：检测 TASK.md 中状态为 ✅ 的任务
2. **创建归档文件**：文件名 = `年月日_时分_任务名称.md`
3. **移动内容**：将任务从 TASK.md 移动到 archive/
4. **重置TASK**：用占位符恢复 TASK.md 到待填写状态

### TASK.md 占位符模板

```markdown
### P0 - 必做任务

⬜ [新增任务占位符]
- 验收标准：xxx
- 相关需求：USER.md 中的 P0-x
```

归档后用新占位符填补空位，用户可继续添加任务。

### 文档分工

| 文件 | 用途 | 执行频率 |
|------|------|----------|
| USER.md | 需求方填写目标 | 仅初始 |
| TASK.md | 执行者任务清单 | 仅初始+更新 |
| Q&A.md | 疑问记录 | 持续 |
| REGRESSION.md | 回归测试（功能是否正常） | 每次 |
| RETROSPECTIVE.md | 复盘记录（经验教训） | 每次 |

## 第二步：添加到 projects_config.json

需要将新项目添加到Supervisor监控目录的 `projects_config.json`：

```json
{
  "version": "1.0",
  "projects": [
    {
      "id": "项目ID",
      "name": "项目名称",
      "path": "路径/async_comm",
      "enabled": true,
      "description": "项目描述"
    }
  ]
}
```

## 第三步：自动创建定时任务

初始化完成后，**技能会自动创建定时任务**（`*/5 * * * *`，每5分钟执行一次）。

使用 CronCreate 工具：
- `cron`: "*/5 * * * *"
- `prompt`: 异步沟通任务执行的完整流程（见下方模板）
- `recurring`: true
- **durable**: true（写入 `.claude/scheduled_tasks.json`，跨会话保留）

```json
{
  "id": "项目ID",
  "cron": "*/5 * * * *",
  "prompt": "异步沟通任务执行 SOP：\n\n## 1. 回归测试\n1. 检查 async_comm/REGRESSION.md 存在\n2. 逐项执行回归测试，更新状态\n3. 如有问题记录到 REGRESSION.md 的修复记录\n\n## 2. 任务执行\n1. 读取 async_comm/TASK.md 获取当前任务清单\n2. 检查是否有已完成任务（状态为✅），如有则：\n   - 创建归档文件：async_comm/archive/YYYY-MM-DD_HHmm_任务名称.md\n   - 将任务内容移动到归档文件\n   - 用占位符填补 TASK.md 空位\n3. 找到第一个状态为⬜的任务\n4. 按验收标准实现该任务\n5. 用web-access验证（必须实际点击）\n6. 完成后更新状态为✅\n7. 输出：完成的任务/验收标准\n\n## 3. 复盘沉淀（关键！）\n1. 检查 async_comm/RETROSPECTIVE.md 存在\n2. 回顾本次任务执行的关键决策和收获\n3. 将经验记录到 RETROSPECTIVE.md 的复盘表\n4. 格式：日期 + 任务阶段 + 经验/教训 + 可复用点\n5. 这些经验供未来同类型工作参考\n\n## 4. Q&A记录\n- 有任何新的想法，记录到 async_comm/Q&A.md\n- 当遇到不确定的问题，记录到「待回答问题」表\n\n## 执行原则\n- 每次执行严格按照 SOP 顺序：回归测试 → 任务执行 → 复盘沉淀 → Q&A\n- 复盘是经验沉淀的关键环节，必须执行\n- 任务内容来自 TASK.md，不要硬编码",
  "recurring": true,
  "durable": true
}
```

**说明**：
- `durable: false`（默认）：只创建lock文件，会话结束后失效
- `durable: true`：创建lock文件 + 写入 `.claude/scheduled_tasks.json`，7天后自动过期

## 核心原则

### 角色分工
- **用户（需求方）**：只写 USER.md，不看 TASK.md 具体内容
- **执行者**：只看 TASK.md，按验收标准执行，不读 USER.md
- **监督者**：检查 USER.md 和 TASK.md 对齐性

### 文档流转
1. 用户在 USER.md 写需求
2. 监督者把需求转成 TASK.md 的验收标准
3. 执行者按 TASK.md 执行，完成后标记✅
4. 执行中的疑问记在 Q&A.md

### 优势
- 用户只需表达"要什么"，不用关心具体怎么做
- 执行者不用担心任务丢失，阅读顺序自上而下
- 监督者保证需求不丢失，执行方向不跑偏
