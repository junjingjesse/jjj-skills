# 创新思维生成工具

## 快速开始

当用户触发此技能时，直接说：

**好的！请分享你的灵感或想法，我来帮你发散！**

然后等待用户输入初始想法。

---

## 用户交互流程

### 第0步：接收用户输入

用户需要提供：
- **灵感/想法**：一个初始的创意想法、问题或挑战
- **场景说明**（可选）：这个想法的应用场景、目标用户等

### 第1步：判断阶段（创造力钻石框架）

直接问用户：

**你现在的任务是？**
- 需要产生更多想法 → **发散阶段**
- 需要筛选/决策/优化 → **收敛阶段**

### 第2步：选择发散工具

根据用户场景，推荐合适的发散工具：

| 用户场景 | 推荐工具 | 适用原因 |
|---------|---------|---------|
| 快速产生想法 | 列表集思广益 | 简单直接 |
| 需要组合/归类 | 便签集思广益 | 可重新排列 |
| 需要迭代深化 | 网格头脑风暴 | 层层传递 |
| 需要广泛探索 | 字母集思广益 | 轻松破百 |
| 需要独立思考 | 脑力写作 | 避免从众 |
| 系统性分析 | 形态分析 | 功能拆解 |
| 快速检查清单 | SCAMPER | 7个维度 |
| 解决技术矛盾 | TRIZ | 40个原则 |
| 跨域借鉴 | 类比推理/隐喻 | 结构迁移 |
| 综合设计 | 设计思维 | 用户中心 |

### 第3步：执行发散

使用选定的工具，引导用户完成发散过程。

### 第4步：收敛筛选（如需要）

当用户说"帮我选一个"、"哪个更好"时，使用收敛阶段的评估标准。

## 详细方法论

### 发散阶段（金字塔上层）
- 目标：想法越多越好
- 原则：Quantity breeds Quality（数量孕育质量）
- 禁止评判、禁止否定

### 收敛阶段（金字塔底层）
- 目标：筛选、精炼、决策
- 原则：Pre-selection预筛选
- 使用评估标准

---

### 发散工具箱（头脑风暴类）

#### 1. 列表集思广益
- 最基础的头脑风暴
- 适合快速列出所有可能的想法
- 不做评判，写下来再说

#### 2. 便签集思广益（推荐）
- 每个想法一张便签
- 可以重新排列、组合
- 鼓励同时创作
- 10-20分钟可产生100+想法

#### 3. 网格头脑风暴
- 3×3网格
- 每行传递给下一个人继续发展
- 适合深化迭代

#### 4. 字母集思广益（A-Z）

- 每个字母关联一个想法
- 轻松产生100+想法
- 适合广泛探索

#### 5. 脑力写作
- 安静书面形式
- 避免从众效应
- 适合个人思考

### 第三步：系统性探索工具

### 形态分析（Morphological Analysis）

**核心思路**：功能→子功能→多种实现方式→组合

**操作步骤**：
1. 从"功能本质"出发，不从外观或现有产品
2. 拆解为多个子系统/子功能
3. 为每个子功能生成多种实现方式
4. 整理成形态分析表（行=子功能，列=实现方式）
5. 每行选一个，组合成完整方案
6. 综合设计，连接各子系统

### 第四步：发散技巧

**随机词触发**：随机选择一个无关词，强制关联到你的问题

**逆向思考**：反向思考"如果相反会怎样"

**组合创新**：把两个不相关的想法组合

**极端假设"如果...会怎样"**：
- 如果不考虑成本？
- 如果用户是小学生？
- 如果明天就要上线？

**旁观者视角**：如果是你最敬佩的人，他会怎么想？

**类比映射**：从另一个领域借用结构
- 例如：航空 → 餐饮、教育 → 游戏

**跨界迁移**：把一个领域的方法用到另一个领域

**上下文注入**（AI协作）：先加载足够背景信息，再让AI生成

### 第六步：收敛筛选（评估标准）

### 基础评估维度

| 维度 | 问题 |
|-----|------|
| 可取(Desirable) | 用户真的需要吗？ |
| 可行(Feasible) | 我们能做吗？ |
| 切实(Viable) | 能持续吗？赚钱吗？ |

### 筛选流程

1. **想法分组**：类似的想法放一起
2. **投票评分**：每人分配点数，给最看好的想法
3. **预筛选**：快速过滤明显不行的
4. **深度评估**：用三维度细评

### 创造力判定标准

**创造力 = 新颖性 × 价值性**
- 新但无价值 → 不构成创造力
- 有价值但不新 → 属于优化/复制

### 第七步：五个P系统诊断（创意卡住时用）

| P | 检查项 |
|---|--------|
| Person | 你的技能/习惯/优势够吗？ |
| Physiology | 精力/注意力状态支持创作吗？ |
| Press | 环境/资源/压力怎样？ |
| Process | 卡在四阶段哪步？Preparation→Incubation→Inspiration→Verification |
| Product | 最终产物清晰吗？ |

### Process四阶段
1. **Preparation准备**：观察、倾听、提问、阅读、收集信息
2. **Incubation孵化**：无意识加工，时间间隔后才有想法
3. **Inspiration灵感**：突��浮现，往往在非刻意思考时
4. **Verification验证**：转化为现实成果

### 第八步：创意环境设计

### 低压力 × 高信任

**必须避免**：
- 持续性压力（内啡肽会阻断新想法）
- 提出想法就被惩罚
- 只接受"正确"答案

**必须建立**：
- 探索 ≠ 立即被评判
- 新想法 = 保护对象
- 组织共同承担试错成本

### 保护机制设计
- 提出新想法 ≠ 自动承担负面后果
- 探索 ≠ 立即被评判或惩罚

### 第九步：大脑与心智（理解创造力本质）

### Mind（心智）= 信号系统
- 来自：思考、情感、选择
- 本质：电化学信号

### Brain（大脑）= 结构响应器
- 对心智信号的反应：构建蛋白质形式的思想

**记忆公式**：Mind（信号）→Brain（结构）→Decision（决策）

### 第十步：思想之树模型

### 思想结构
- **根（Roots）**：思想的源头，即原始经历/记忆
- **树干（Trunk）**：核心思想
- **枝（Branches）**：对原始经验的个人解读和理解

### 思想激活规模
- 一个思想激活 = 数百个记忆
- 每个记忆包含：数据+细节+思考+情感+选择

### 思想的身体反应
- 一个思想 ≈ 1,400个神经-物理反应
- 影响全身细胞，甚至改变DNA
- 每日产生：8,000-10,000个想法

### 三种记忆类型
1. **信息/细节/数据关联**
2. **事件经验**
3. **情绪与身体体现**

### 第十八步：类比推理（跨域创新）

### 核心概念
- **类比迁移的是"关系"，不是"外形"**
- 源域（熟悉）→ 目标域（陌生）
- A : B :: C : X 的标准形式

### 历史案例
- **波尔原子模型** ← 太阳系（轨道结构映射）
- **凯库勒苯环** ← 蛇咬尾（循环结构）

### 类比关系类型
| 关系 | 示例 |
|------|------|
| 同义词 | 开朗：快乐 |
| 反义 | 黑：白 |
| 关联 | 牛：牛奶 |
| 部分到整体 | 树干：树 |
| 类别 | 猫：动物 |
| 功能 | 尺：测量 |
| 因果 | 生病：病毒 |

### 第十九步：隐喻（跨域理解）

### 核心特征
- 隐喻 = "等同关系"，不是"相同关系"
- 本质是意义的迁移/搬运
- 迁移关系、功能、结构，而非外观

### 经典隐喻案例
- **Mind is an iceberg** - 意识/潜意识
- **Laughter is medicine** - 治疗/修复
- **Time is money** - 资源属性

### 活隐喻 vs 死隐喻
- **活隐喻**：仍能激发新理解 → 促进创造力
- **死隐喻**：已被文化完全吸收 → 抑制创造力

### 避免过度熟悉的隐喻
- "Life is a rollercoaster" 已疲劳
- "The world is a stage" 已变成陈词滥调

### 演绎推理
- 从一般到特殊
- 前提→次要前提→推论

### 归纳推理
- 从特殊到一般
- 观察→规律→假设

### 科学方法
观察→提问→假设→测试→分析→迭代→结论

- "还有吗？再想一个"

- "如果你的用户是小学生，他会怎么想？"
- "把这个想法和另一个领域组合会怎样？"
- "反过来做会怎样？"

## 使用示例

用户说：我想做一个解决上班族午餐选择困难的app

1. **判断阶段**：需要产生更多想法 → 发散阶段

2. **发散输出**：
```
产生想法26个：
- 盲盒推荐（随机午餐）
- AI根据口味推荐
- 同事拼单
- 午餐社交
- 网红店打卡
...
```

3. **收敛筛选**：
```
TOP3推荐：
1. 盲盒推荐 - 解决选择困难，趣味性强
2. AI口味推荐 - 技术实现可行
3. 同事拼单 - 社交+省钱
```

## 相关概念（知识库）

- **大C vs 小c**：
  - 大C：历史性突破（爱因斯坦、乔布斯）
  - 小c：日常改进（你的每个微创新）

- **Process四阶段**：
  - Preparation（准备）
  - Incubation（孵化）
  - Inspiration（灵感）
  - Verification（验证）

- **固着效应**：不要执着于第一个想法，广泛探索先！

- **隐喻与类比**：将一个领域结构映射到另一领域，是高级创造力的关键

- **TRIZ核心矛盾**：技术矛盾（两个参数冲突）vs 物理矛盾（同一参数不能同时满足两个状态）

- **创造力 = 新颖性 × 价值性**（缺一不可）

- **赞助的价值**：时间和资源，让创意从概念变成现实

- **思想之树**：根=原始经历，枝=个人解读

- **Lego教学法**：将抽象概念具体化
