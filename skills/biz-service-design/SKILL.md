# 服务设计思想实验

## 输入格式

用户提供：
- 商业想法/服务概念
- 目标顾客是谁
- 想解决什么问题

## 分析框架

### 第一步：顾客旅程痛点分析

请用户回答：
1. 顾客要完成什么任务？(Job to be Done)
2. 顾客现在怎么做？
3. 过程中卡在哪儿？（痛点）
4. 顾客勉强接受了什么？（犧牲）

### 第二步：服务蓝图构想

请用户构想：
- 顾客可见的前台是什么？
- 后台需要什么支撑？
- 哪些环节可能出错？
- 实体验证（Physical Evidence）有哪些？

### 第三步：收益模型

请用户思考：
- 顾客愿意为什么付费？
- 可以差别定价吗？
- 时段/功能/渠道差异？

### 第四步：平台可能性

请用户想：
- 需要联结哪几方？
- 网络效应在哪边？
- 交叉补贴怎么设计？

## 输出格式

每次分析请给出：

1. **顾客旅程图**：时间轴+接触点+痛点标记
2. **服务蓝图草图**：前台/后台/支援
3. **3个机会点**：可优化的环节
4. **3个风险点**：可能失败的地方
5. **1个差异化建议**：和竞品不一样的地方

## 激发问题

分析过程中，随时追问：
- "顾客真的在乎这个吗？"
- "有没有更简单的做法？"
- "这个体验能记住吗？"
- "如果不收费还能做到吗？"
- "线下能做到吗？必须线上？"
